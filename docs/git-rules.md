## Commit Message Convention

시맨틱 커밋 메시지 규칙을 사용한다.
모든 커밋 메시지는 한글로 작성한다.

- 포맷: `<type>(<scope>): <short summary>`
- `<scope>`는 선택 사항
- 요약은 짧고 간단하게 현재 시제로 작성

예시:
- `feat: 모자에 귀여워지는 효과 추가`

타입 가이드:
- `feat`: 새로운 기능 추가
- `perf`: 성능 개선
- `fix`: 버그/이슈 해결
- `ci`: CI 설정/스크립트 변경
- `cd`: CD 설정/스크립트 변경
- `ci/cd`: CI와 CD 설정을 함께 변경
- `docs`: 문서 변경
- `style`: 포맷팅, 세미콜론 등 코드 동작 변화 없는 변경
- `refactor`: 코드 리팩터링 (동작 동일)
- `test`: 테스트 코드 추가/변경
- `chore`: 코드 수정 없는 설정 변경 또는 기타 자잘한 변경

이슈 번호 예시:
- `fix(#1): 고양이가 나타나지 않던 문제 해결`

주의:
- `fix`를 포함한 커밋 메시지는 이슈를 닫을 수 있으므로 의도적으로 사용한다.

주요 변경 강조:
- `!`를 사용해 브레이킹 변경 또는 중요한 변경을 강조한다.
- 예시:
  - `feat!: 회원가입 인증 메일에 고양이 사진을 포함하지 않습니다.`
  - `feat(auth)!: 회원가입 인증 메일에 고양이 사진을 포함하지 않습니다.`
  - `fix(#99)!: 고양이 보안 문제 해결`

## Pre-commit Checks

모든 커밋 전에 반드시 `npm run lint`를 실행하고, 오류가 없을 때만 커밋한다.

권장 순서:
1. `npm run lint`
2. (필요 시) `npm run build`
3. `git commit`
